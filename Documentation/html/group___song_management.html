The backend for songs and lessons from an audio perspective.

Terminology

Melody

A type of Song that contains only pitches.

Drum Loop

A type of Song that contains only drums. These are looped when played.

See also
VirtualInstrumentManager::PlayDrumLoopEvent

Combined

A type of Song that has both pitches and drums.

Song File Format

There are three types of Songs which can be stored in a file. Which type the Song is will affect how it is saved. For all song types, the files are saved in Assets/StreamingAssets/Songs/.

Note
The values from Music enums are typecast to integers for saving. Ex: Music.PITCH.C4 is saved as 48.

File Format for a Melody

If the song is a melody, then the file is saved as "MELODY_songName.song". The format of the file is

  • First line: "Name of song"
  • Second Line: "songType;defaultBPM;beatsPerMeasure;baseBeat"
  • Rest of lines: "pitch0,pitch1,...,pitchN;noteLength;offsetFromPrevNote;melodyVelocity"
  • Example from "MELODY_Melody Example.song": 
    Melody Example
1;111;4;6
36,40,43;10;12;100
41;2;5;70
52;2;4;70
34;2;4;70
36,40;6;4;70
36,43;6;4;70
36,41;6;6;70
52,57,60,48;10;4;70
55,52,48,36;10;6;70

File Format for a Drum Loop

If the song is a drum loop, then the file is saved as "DRUMLOOP_songName.song". The format of the file is:

  • First line: "Name of song"
  • Second Line: "songType;defaultBPM;beatsPerMeasure;baseBeat"
  • Rest of lines: "drum0,drum1,...,drumN;offsetFromPrevNote;drumVelocity"
  • Example from "DRUMLOOP_Drum Loop Example.song": 
    Drum Loop Example
2;120;4;6
0,6;4;100
0;4;100
6,2,0;4;100
0;4;100
10,0;4;100
0;2;100
1;2;100
0;2;100
6,2;2;100
0;4;100

File Format for a Combined Song

If the song has both melody and percussion, then it will be saved as "SONG_songName.song". The format of the file is:

  • First line: "Name of song"
  • Second Line: "songType;defaultBPM;beatsPerMeasure;baseBeat"
  • Rest of lines: "pitch0,pitch1,...,pitchN|drum0,drum1,...,drumN;noteLength;offsetFromPrevNote;melodyVelocity|drumVelocity"
    Note
    If pitches or drums don't exist for a note, then "null" is used in the first section of lines 3+ and the velocity for the missing type is set to 0.
  • Example from "SONG_Combined Example.song": 
    Combined Example
3;203;4;6
48,52,55,58|0,8;6;12;100|100
null|8,0;12;4;0|100
48,52,55,60|2,8,4;4;4;100|100
50,53,58|0,8;6;4;100|100
null|8,0;12;4;0|100
57|0,8;4;4;100|100
48,60,64,52,55|0,2,4,8;4;4;100|100
48,60,52,64,67,55,72|0,2,4,8;4;4;100|100
